# EchoProfile 思想回响素材规范

这份文档总结 `EchoProfile` 项目里「思想回响」素材的实际使用要求，基于当前的 EchoProfile 产品形态整理。

目标不是做百科全书，而是保证人物素材在当前 UI 里：

- 显示统一
- 多语言可用
- 卡片高度可控
- 适合共振推荐场景

## 一、每位科学家需要的素材字段

当前每位科学家至少应包含这些字段：

```json
{
  "slug": "herbert_simon",
  "name": "Herbert A. Simon",
  "localized_names": {
    "zh": "司马贺",
    "ja": "localized Japanese name",
    "ko": "localized Korean name"
  },
  "portrait_url": "/scientist-portraits/herbert_simon.jpg",
  "quote_zh": "信息的丰富，造成了注意力的贫乏。",
  "quote_en": "A wealth of information creates a poverty of attention.",
  "core_traits": "工程理性、跨域组织、决策框架、有限理性",
  "thinking_style": "……",
  "temperament_tags": "……",
  "temperament_summary": "……",
  "loading_copy_zh": "……",
  "loading_copy_en": "……",
  "bio_zh": "……",
  "bio_en": "……",
  "achievements_zh": ["……", "……", "……"],
  "achievements_en": ["……", "……", "……"]
}
```

## 二、照片要求

### 1) 文件位置

- 图片实际资源放在 `public/scientist-portraits/`
- 数据里通过 `portrait_url` 引用，例如：
  - `"/scientist-portraits/herbert_simon.jpg"`

### 2) 画面要求

- 以单人肖像为主
- 头部和上半身优先，尽量不要群像
- 适合裁成竖版
- 当前 UI 会自动做灰度显示，所以彩色不是必须

### 3) 比例建议

卡片中图片容器使用的是 `aspect-[4/5]`，所以建议：

- 优先准备接近 `4:5` 的竖版图
- 如果原图不是这个比例，也要保证人物主体位于中间，便于裁切

### 4) 格式建议

当前仓库中实际混用了：

- `.jpg`
- `.jpeg`
- `.png`

建议后续尽量统一为：

- 优先 `jpg`
- 若原图透明背景或修复效果更好，可用 `png`

### 5) 清晰度建议

- 至少保证卡片里放大后不糊
- 建议短边不低于 `400px`
- 不需要超大原图，网页展示优先

## 三、名字要求

### 1) 英文主名

- `name` 必须保留英文名
- 它既是默认展示名，也是英文维基链接的后备标题

### 2) 多语言名字

当前项目已经支持：

- 中文：`localized_names.zh`
- 日文：`localized_names.ja`
- 韩文：`localized_names.ko`

规则如下：

- 中文界面优先显示 `localized_names.zh`
- 日文界面优先显示 `localized_names.ja`
- 韩文界面优先显示 `localized_names.ko`
- 英文界面显示 `name`
- 如果对应语言没有本地化名字，则回退到英文名

### 3) 中文名来源

中文名应优先以中文 Wikipedia 条目名称为准。

例如：

- `Herbert A. Simon` -> `司马贺`
- `John von Neumann` -> `约翰·冯·诺伊曼`
- `Claude Shannon` -> `克劳德·香农`

不要随意混用：

- 音译版本
- 港台版本
- 自行意译版本

除非你明确决定要做地区化分支。

## 四、名言要求

### 1) 当前 UI 的实际情况

名言在卡片里会单独放进一个灰底引用块中展示：

- 不再显示引用图标
- 会自动去掉外层包裹引号
- 内部术语引号会保留

### 2) 长度建议

当前数据统计大致是：

- `quote_zh` 平均长度约 `25` 字
- 当前最大值约 `42` 字

为了避免卡片过高，建议控制在：

- 理想长度：`12-28` 字
- 可接受长度：`28-40` 字
- 尽量不要超过：`40` 字

### 3) 内容风格

优先选择：

- 一句能代表其思维方式的话
- 适合在卡片里独立阅读
- 信息密度高、可记忆

尽量避免：

- 一大段长论述
- 需要上下文才能理解的句子
- 过多从句和解释性插入

### 4) 当前已经证明过长的类型

以下类型会让卡片显著变高：

- 两句以上合并成一条
- 带完整定义、因果展开、长转折
- 类似 Herbert Simon 旧版本那种完整注意力经济学段落

更好的做法是：

- 换成同人物的更短经典句
- 或只保留长句中最核心的第一判断

## 五、人物简介要求

### 1) 简介用途

`bio_zh / bio_en` 不是百科全文，而是给共振卡片服务的「短简介」。

它应当帮助用户快速知道：

- 这个人是谁
- 为什么重要
- 他/她最核心的学术或工程定位是什么

### 2) 推荐长度

当前数据 `bio_zh` 的平均长度约为 `75` 字。

建议控制在：

- 理想长度：`70-110` 字
- 最少不要低于：`60` 字
- 超过 `120` 字就容易让卡片显得太满

### 3) 推荐结构

最好写成两句：

- 第一句：人物定位
- 第二句：核心贡献 / 历史意义

推荐模板：

```text
X 国/地区的某领域科学家，以某种能力或方法著称。
他/她在 A、B、C 上的工作，如何影响了后来的某个方向。
```

### 4) 不推荐写法

避免把简介写成：

- 单纯履历堆砌
- 生平流水账
- 过于冷的百科句子
- 与“代表成就”重复过高

### 5) 和“代表成就”的分工

- `bio` 负责解释“这个人是谁、为什么重要”
- `achievements` 负责列举“最典型的 2-3 个成果”

不要把所有成果都塞进 `bio`。

## 六、代表成就要求

### 1) 数量建议

当前卡片设计最适合：

- `3` 条最佳
- `2-4` 条可接受

太少会显空，太多会让右侧卡片过高。

### 2) 单条长度建议

每条成就建议控制在：

- `8-18` 字左右
- 尽量一句话说清楚

示例：

- `创立信息论`
- `提出 lambda 演算`
- `主持建成首个可控核反应堆`

### 3) 写法建议

优先写：

- 公认成果
- 高识别度成果
- 与该人物最强相关的成果

尽量避免：

- 模糊评价句
- 过长解释句
- 和简介重复

## 七、共振标签要求

`core_traits`、`temperament_tags`、`resonance_axes` 最终会被展示成标签。

建议：

- 单个标签尽量短
- 2-6 个字最合适
- 可读、可扫视、不要像完整句子

示例：

- `工程理性`
- `系统统筹`
- `有限理性`
- `实验手感`

## 八、维基百科链接要求

当前每张科学家卡片都支持显示 `Wikipedia / 维基百科` 链接：

- 主共振卡片：放在「人物简介」卡片底部
- 次共振卡片：也显示链接
- 近期状态卡片：也显示链接

跳转规则：

- 中文界面优先跳中文维基
- 日文界面优先跳日文维基
- 韩文界面优先跳韩文维基
- 没有对应语言时回退到英文维基

因此素材层需要保证：

- `name` 可用于英文维基标题
- `localized_names` 尽量与对应语言 Wikipedia 页面标题一致

## 九、当前最适合的素材标准

如果你后续还要继续加新科学家，建议按这个标准准备：

### 必备

- 1 张竖版肖像图
- 英文名
- 中文名
- 1 句短名言
- 2 句简介
- 3 条代表成就
- 一组短标签

### 推荐标准

- 照片：`4:5` 竖版，人物居中
- 名言：`12-28` 字最佳
- 简介：`70-110` 字最佳
- 成就：`3` 条最佳
- 单条成就：`8-18` 字最佳

## 十、后续补人物时的检查清单

- 照片是否适合裁成 `4:5`
- `portrait_url` 是否指向 `public/scientist-portraits/`
- 是否有英文名 `name`
- 是否有中文名 `localized_names.zh`
- 中文名是否优先参考中文 Wikipedia
- 名言是否过长
- 简介是否至少两句、长度足够
- 成就是否控制在 `2-4` 条
- 成就是否和简介严重重复
- 中文、英文内容是否都存在

## 十一、建议的维护原则

- 优先统一风格，而不是机械追求百科完整
- 优先可展示性，而不是信息量最大化
- 优先人物“思维气质”相关素材，而不是泛泛生平
- 如果某位科学家的经典名言太长，宁可换一句更短、更有辨识度的
